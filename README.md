# module_tests

Ce module se divise en deux parties 
## Partie 1 : 
Initiation aux méthodes de tests.

Création de deux projets simples :
  * **Calculator** : une classe simple d'outils de calcul. (+ , - , * , / );
  * **InjectionDependances** : une application servant à saluer l'utilisateur en fonction de la date /heure locale. Utilisation des Interfaces pour pouvoir switcher l'heure locale par une heure factice pour les tests.

Objectifs de la première partie :  
    * Connaître les différents types de tests et leurs intérêts  
    * Définir une stratégie de test  
    * Apprendre à mettre en place des tests unitaires  
    * Appréhender une base de code « legacy » et la faire évoluer  
    * Pratiquer le TDD
    
## Partie 2 : (pas encore réalisée)
Objectifs de la seconde partie :   
    * Consolider les compétences en tests unitaires  
    * Définir une stratégie de test  
    * Tester une application de bout en bout
